## 一、设计理念

### 核心概念

**"冰冷的秩序，温热的灵魂"**

法庭是理性的殿堂，但每一个案件都在拷问人性。UI 必须呈现这种张力——

- **外层**：冷峻、精密、全息感（赛博朋克 + Blade Runner 的视觉语言）
- **内核**：留白、呼吸感、情绪留痕（Minority Report 的预知悬浮界面）

### 设计原则

| 原则 | 说明 |
|------|------|
| **全息优先** | 所有界面元素带有全息投影质感——半透明、边缘发光、扫描线 |
| **数据即叙事** | UI 不只是功能层，也是世界观的一部分（判决书、证据档案本身就是剧情载体） |
| **情绪留痕** | 关键选择后，界面留下"情绪残影"——淡淡的色彩偏移，暗示立场变化 |
| **极简非装饰** | 没有多余的装饰图案，每一个元素都在传递信息或情绪 |

### 视觉关键词

```
Holographic · Cold Blue · Scan Lines · Glitch · 
Glass Morphism · Floating Data · Judicial Gravity · 
Neural Interface Feedback
```

## 二、字体系统

### 2.1 字体选型

| 用途 | 字体 | 字重 | 说明 |
|------|------|------|------|
| **主标题（Logo / 章节名）** | Orbitron | 700 / 900 | 几何感强，未来法庭的庄严感 |
| **界面元素（按钮 / 标签）** | Rajdhani | 500 / 600 | 窄体，科技感，信息密度高 |
| **正文对白（中文）** | 思源黑体 / Source Han Sans | 400 / 500 | 清晰易读，中文渲染优秀 |
| **正文对白（英文/数字）** | Exo 2 | 400 | 与思源黑体协调，科技中性 |
| **证据/档案文本** | JetBrains Mono | 400 | 等宽字体，营造数据档案感 |
| **AI 角色对白** | Orbitron | 400 | 与人类角色区分，AI 的"机器感" |
| **回声（Echo）特殊文本** | Share Tech Mono | 400 | 故障感、失真感 |

### 2.2 字号层级

```
显示级标题 (Logo / 章节大标题):    64-96px / Orbitron 900
界面标题 (界面顶部标签):            28-36px / Rajdhani 600
按钮文字:                          20-24px / Rajdhani 500
正文对白:                          26-30px / 思源黑体 400
证据档案正文:                      20-22px / JetBrains Mono 400
辅助说明 (提示 / 时间戳):          14-16px / Rajdhani 400
```

### 2.3 特殊文字效果

- **全息文字**：文字底部带蓝色辉光 `text-shadow: 0 0 8px rgba(0,200,255,0.6)`
- **故障文字**（回声专用）：周期性 RGB 错位 + 水平撕裂
- **扫描线文字**：文字表面叠加横向扫描线纹理
- **判决书文字**：打字机效果，逐字浮现，每字带轻微闪烁

## 四、通用视觉元素

### 4.1 全息面板

所有界面面板采用"全息玻璃"质感：

- 背景：`rgba(10, 14, 26, 0.85)` 半透明深色
- 边框：1px 实线 `rgba(0, 200, 255, 0.3)`，四角带 L 形装饰角标（青蓝色直角括号）
- 内发光：`box-shadow: inset 0 0 30px rgba(0,200,255,0.05)`
- 扫描线纹理：表面叠加极淡的横向扫描线
- 边缘故障：随机位置的像素级 RGB 错位（极轻微）

### 4.2 角标装饰

面板四角的 L 形角标是本作的视觉签名：

```
┌─                          ─┐


└─                          ─┘
```

- 颜色：全息蓝 `#00C8FF`
- 粗细：2px
- 尺寸：20×20px
- 动画：呼吸式微弱明暗变化（2 秒周期）

### 4.3 数据流装饰

界面边缘可装饰流动的数据流——

- 一串快速滚动的十六进制 / 二进制字符
- 颜色：`rgba(0, 200, 255, 0.15)`（极淡）
- 速度：缓慢，不抢眼
- 位置：屏幕左右边缘竖向滚动，或顶部横向滚动

### 4.4 粒子效果

- 全息蓝光粒子缓慢上浮（如尘埃在光柱中）
- 密度：低（每屏约 15-20 粒）
- 大小：1-3px
- 透明度：0.2-0.5 随机
- 用于主菜单、法庭界面、判决界面

### 4.5 神经接口指示器

主角度色专属 UI 元素，位于对话界面右下角——

- 一个小型圆形波形图
- 实时反映"感知到的 AI 情绪"
- 颜色随情绪变化：平静（蓝）、愤怒（红）、悲伤（紫）、喜悦（金）
- 仅在对话中显示

## 八、读档界面

### 8.1 布局

读档界面与存档界面**结构完全一致**，区别如下：

| 差异点 | 存档界面 | 读档界面 |
|--------|---------|---------|
| 标题 | "存档 · SAVE" | "读档 · LOAD" |
| 空槽位 | 可点击存档 | 不可点击，显示为禁用态（灰色） |
| 已有存档按钮 | [存档] [删除] | [读取] [删除] |
| 标题色 | 全息蓝 | 冰蓝（略浅，区分功能） |
| 悬停动作 | 存档写入动画 | 读档读取动画（进度条 + "数据恢复中"文字） |

### 8.2 读档动画

1. 点击 [读取] → 槽位边框高亮
2. 全屏淡出至黑（0.3s）
3. 扫描线从上至下扫过（0.5s）
4. "数据恢复中..." 文字居中显示，JetBrains Mono，带打字效果
5. 进度条 0% → 100%（1s）
6. 淡入游戏场景

### 9.2 设计说明

| 元素 | 说明 |
|------|------|
| **分栏** | 三个并列全息面板：显示 / 音频 / 文字。底部一个横跨全宽的辅助功能面板 |
| **单选框** | `◉` 选中态为全息蓝实心，`○` 未选中态为钢灰空心。切换时有 0.15s 缩放动画 |

## 十六、响应式与适配

### 16.1 目标分辨率

| 优先级 | 分辨率 | 说明 |
|--------|--------|------|
| 主要 | 1920×1080 | 设计基准 |
| 次要 | 1280×720 | 低分辨率适配 |
| 兼容 | 2560×1440 | 高分屏适配 |

### 16.2 适配策略

- 所有 UI 元素使用相对定位（锚点 + 比例）
- 字体大小支持缩放（设置中可调）
- 16:9 宽高比为主，超宽屏左右留黑边 + 数据流装饰填充
- 4:3 等非标准比例，上下留黑边 + 扫描线填充

### 16.3 可访问性

- **色弱模式**：立场指示色增加形状区分（▲◆●），不仅依赖颜色
- **字体放大**：1.0x / 1.25x / 1.5x 三档
- **减少动效**：关闭粒子、扫描线、故障效果，保留必要过渡
- **高对比模式**：提升文字与背景对比度，减弱全息透明度
